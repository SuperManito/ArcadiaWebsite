## 更新 CLI 配置

<POST>/config/cli</POST>

支持部分字段更新，仅传入需要修改的字段即可。保存成功后自动重新生成 `.arcadia_cli_config.sh` 配置批量声明脚本。

### 请求

<TypeTable
  type={{
    REMOVE_LOG_DAYS_AGO: {
      description: '日志保留天数（正整数字符串）',
      type: 'string<number>',
    },
    ENABLE_UPDATE_EXTRA: {
      description: '是否启用自定义更新脚本',
      typeDescription: '"true" 启用，"" 禁用',
      type: 'string',
    },
    ENABLE_UPDATE_EXTRA_SYNC_FILE: {
      description: '是否启用自定义更新脚本远程同步',
      typeDescription: '"true" 启用，"" 禁用',
      type: 'string',
    },
    UPDATE_EXTRA_SYNC_FILE_URL: {
      description: '自定义更新脚本的远程同步地址',
      type: 'string',
    },
    ENABLE_INIT_EXTRA: {
      description: '是否启用自定义初始化脚本',
      typeDescription: '"true" 启用，"" 禁用',
      type: 'string',
    },
    ENABLE_TASK_BEFORE_EXTRA: {
      description: '是否启用任务运行前自定义脚本',
      typeDescription: '"true" 启用，"" 禁用',
      type: 'string',
    },
    ENABLE_TASK_AFTER_EXTRA: {
      description: '是否启用任务运行后自定义脚本',
      typeDescription: '"true" 启用，"" 禁用',
      type: 'string',
    },
    ENABLE_AUTO_DELETE_REMOTE_FILE: {
      description: '是否自动删除远程执行后的本地脚本文件',
      typeDescription: '"true" 启用，"" 禁用',
      type: 'string',
    },
    ENABLE_CUSTOM_NOTIFY: {
      description: '是否启用自定义推送通知模块',
      typeDescription: '"true" 启用，"" 禁用',
      type: 'string',
    },
    RUN_DELAY_MAX_SECONDS: {
      description: '随机延迟最大秒数（非负整数字符串）',
      type: 'string<number>',
    },
  }}
/>

```json title="示例"
{
  "REMOVE_LOG_DAYS_AGO": "14",
  "ENABLE_UPDATE_EXTRA": "true"
}
```

### 响应

无数据，仅返回操作状态。

<Callout type="info">
  CLI 配置统一存储于数据库 `config` 表（`module = 'cli'`）中，键名为去除 `CLI_CONFIG_` 前缀的字段名。每次保存后自动生成位于用户配置文件目录下的 `.arcadia_cli_config.sh` 脚本，该脚本会在底层 Shell 命令执行时自动加载。
</Callout>
